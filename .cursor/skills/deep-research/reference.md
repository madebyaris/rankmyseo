# Deep research — quick checklist

Reuse between sessions; copy into notes only when tracking long arcs.

## Intake

- [ ] Freeze version or time scope
- [ ] Classify question (compare, diagnose, explain, procedural, forecast)
- [ ] Name non-goals
- [ ] Define adversarial success test (what convinces a sceptic)
- [ ] Draft a brief research plan: scope, anchors, counter-evidence pass, stop condition

## Retrieval passes

| Pass | Intent | Done when |
| --- | --- | --- |
| 1 | Anchor primaries per sub-question | Each matrix cell has at least one authoritative source |
| 2 | Independence and tension find | Divergence surfaced or absence explained |
| 3 | Compress noise | Queries tightened to discriminators |
| 4 | Uncertainty audit | Known-unknown list is explicit |

## Synthesis QA

- [ ] Strong recommendations carry **[P]** or **[S]** basis, or are **[H]** / **[A]**
- [ ] Deprecated, preview, or beta behaviour flagged
- [ ] Sensitive security claims checked against more than one weak channel when needed
- [ ] Benchmark methodology reproducible or limitation stated
- [ ] Citations directly support adjacent claims
- [ ] Negative evidence search completed for high-impact recommendations
- [ ] Reader gets a repeatable verification path

## Stop rule

Cease retrieving when the last pass only duplicated primary paraphrases or failed to add discriminative facts. Summarise and surface uncertainty instead.
